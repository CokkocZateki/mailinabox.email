Choosing a Domain Name

Every domain name is within a TLD, or top-level domain. This page lists some of the quirks Mail-in-a-Box users have found in TLDs that might impact whether you can successfully set up Mail-in-a-Box.

Good TLDs

The following TLDs have worked for us:

  • .info
  • .me

Known problems

The following TLDs have had issues:

  • .us (couldn't set DNSSEC DS record at Godaddy)
  • .de (requires two distinct nameservers)
  • .at (requires two distinct nameservers)
  • .email (DNSSEC DS record requires RSA/SHA-256 specifically)

Other TLDs

For other TLDs, unfortunately we don’t yet know.

If you will register your domain name with Gandi, check their list of TLDs that support DNSSEC. DNSSEC is optional on Mail-in-a-Box, but it’s nice to have.

Please submit a patch by forking this repository if you find anything out.
